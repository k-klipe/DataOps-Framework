check_name: название проверки
owner: логин владельца проверки
alert_users: теги юзеров, который будут призываться при отправки уведомления в чат
check_type: тип проверки percentage_deviation/standart_deviation/compare_threshold/check_isolation_forest/check_prophet_forecasting/check_psi_stability
schedule: расписание проверки в формате cron
queries: в этом блоке хранится информация о запросах в базы данных. Принимает значение source/target
      db: Тип БД. Возможные значения vertica/clickhouse
      connection_id: id с json, в котором лежит пароль к бд
      query: ссылка на файл с sql запросом. Значения source.sql/target.sql
      date_column: название колонки с датой из запроса
      value_column: название колонки со занчением из запроса
threshold: порог срабатывания алерта
abs: Флаг сравнения по модулю. Если True –от расчётного значения проверки берётся модуль для сравнения с порогом
larger: Если True – работает в обратном направлении, то есть нормальной ситуацией считается если контрольное значение выше порога
text: Текст, который будет отправляться в канал при срабатывании алерта
send_always: Если True- сообщение отправляется при каждом запуске проверки, если False- только в случае пробития порога
send_graph: Флаг отправки в тред графика дата-значения
send_details: Флаг отправки в тред детальной информации проверки. Присылает все значения, получаемые из запросов sql
skip: Принудительный пропуск проверки, если True – поверка никогда не запускается
